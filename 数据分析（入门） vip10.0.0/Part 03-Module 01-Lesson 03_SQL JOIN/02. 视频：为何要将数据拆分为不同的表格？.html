为何使用不同的表格？

数据库规范化

在创建数据库时，一定要思考下将如何存储数据。这称为规范化，是大多数 SQL 课程的一个重要组成部分。如果你负责设置新的数据库，则需要详细了解数据库规范化。

数据库规范化需要考虑以下三个要点：

  1. 表格存储了逻辑分组的数据吗？
  2. 我能在一个位置进行更改，而不是在多个表格中对同一信息作出更改吗？
  3. 我能快速高效地访问和操纵数据吗？

这篇文章详细讲解了上述内容。

但是，大多数分析师处理的数据库已经设置了必要的属性。作为数据分析师，你不需要太思考数据规范化。你只需能够从数据库中获取数据，从而能够绘制图表。这将是这节课的侧重点。
