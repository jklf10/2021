04. [Windows] 配置 Git Bash 以运行 Python

对于使用 Git Bash 的 Windows 用户：

如果你在 Windows 上并选择使用 Git Bash，则需要运行一些命令来配置它以运行 Python 和 Anaconda。我们将通过创建 .bashrc 文件来完成此操作。

打开 Git Bash 终端并按照以下步骤操作。

步骤 1: 输入 cd 到你的主目录。

这一步很重要。默认情况下，Git Bash 不会在此目录中打开，你需要在此处创建 .bashrc 文件。

步骤 2：输入 pwd 以获取主目录的路径。

pwd 为你提供当前目录的路径，如果你执行了步骤 1，则该路径应该是你的主目录。下一步操作会需要此目录。

我的主目录的路径是 /c/Users/Juno。

步骤 3：在 .bashrc 文件中将 Python 和 Anaconda 添加到 PATH。

对于此步骤，你将需要计划安装 Anaconda 的完整文件路径。如果你在 Anaconda 安装过程中使用了默认设置，那么这应该是你从上一步获得的路径，最后附加了 /Anaconda3。举个例子，在我这里就是 /c/Users/Juno/Anaconda3。

然后，在终端中输入以下命令，将 [YOUR_PATH] 替换为 Anaconda 安装的路径。

echo 'export PATH="$PATH:[YOUR_PATH]:[YOUR_PATH]/Scripts"' >> .bashrc

这为 .bashrc 文件添加了两个路径，告诉 Git Bash 在哪里可以找到执行 Python 和 Anaconda 所需的脚本。同样，完成此步骤后，确保你在主目录非常重要。

步骤4：在 .bashrc 文件中为 Python 添加 alias。

接下来，输入此命令告诉 Git Bash 在哪里可以找到 Python 可执行文件。

echo 'alias python="winpty python.exe"' >> .bashrc

这是在 Git Bash 中运行 Python 的用户所需的额外步骤。这会向你的 .bashrc 文件添加一个命令，当你在终端输入 python 时，它会运行 winpty python.exe。

步骤5：执行 .bashrc 中的命令

现在，你已将必要的命令添加到 .bashrc 文件中，请运行以下行来执行该文件。

source .bashrc

或者，你可以关闭目前的窗口，并打开一个新的 Git Bash 窗口，这将完成同样的事情。 每次 Git Bash 启动时都会执行 .bashrc。

步骤 6：测试运行

运行以下命令以确保你可以访问 conda、Python 和 Python 解释器。 如果你正确地执行这些步骤，你现在应该能够在 Git Bash 中运行 Python！

输入 exit() 以退出 Python 解释器。
