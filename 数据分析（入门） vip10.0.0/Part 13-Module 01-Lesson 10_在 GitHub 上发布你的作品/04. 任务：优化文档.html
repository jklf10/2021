没有一种方法可以告诉我们如何撰写 README 文件，但是有一些常规约定可以参考。

  1. 对项目的描述或仓库所含内容的简介。
  2. 该项目可能需要的软件、固件和硬件列表。
  3. 软件和固件的安装说明。
  4. 项目中包含的文件列表。
  5. 版权和许可信息。
  6. 对帮助你创建此项目的任何资源或博客的致谢。
  7. 已知的 bugs
  8. 更新和修订日志。

你可以在此处查看有关此 GitHub 资源上的 README 文件的一些有用文档。
