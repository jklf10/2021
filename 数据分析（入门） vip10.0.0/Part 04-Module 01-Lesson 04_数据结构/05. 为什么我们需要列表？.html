05. 为什么我们需要列表？

为什么我们需要列表？



我们来谈谈为什么我们需要像列表这样的数据结构，以及什么时候使用它。 我们将通过华尔街的一个例子进行讨论。

在纳斯达克交易所上市的公司有每个公司名称的股票代码或缩写。 例如 Alphabet, Inc. 的股票代码是GOOGL。

现在想象一下，你拥有一家公司的股票，比如微软 Microsoft，并希望能够打印出股票的股票代码。 因为它是一个值，所以可以将它存储在变量microsoft中，并为其赋值为MSFT。 像这样：

microsoft = MSFT

嗯，这很方便！ 因此，现在当你想要为你持有股票的公司打印股票代码时，就可以直接使用print命令。
```
print(microsoft)

MSFT ```

现在假设你是一名投资基金经理，并且希望打印出你在指数基金中拥有的股票（或持股）（例如，Vanguard Institutional Index Fund)。指数基金包括大量公司的股票（也称为持股）。可以看到，先锋机构指数基金已持有506控股！

按照之前的做法，使用单个字符串打印所有506个持股的话，需要506个字符串。 这并不是我们想要的！因为我们需要记住每个字符串的名称才能打印出来。

你还必须考虑如何在相同的索引基金下对506个字符串进行分组。 太不方便啦！

这就是体现数据结构之美的地方！ 你可以使用一个 list。

由于指数基金也有股票代码，你可以使用它作为列表的名称，比如说例子里的VINIX，并将所有持股的股票代码符号添加到该 list 中。 让我们列出Vanguard Institutional Index Fund中的最大持股的几个。

VINIX = ['C', 'MA', 'BA', 'PG', 'CSCO', 'VZ', 'PFE', 'HD', 'INTC', 'T', 'V', 'UNH', 'WFC', 'CVX', 'BAC', 'JNJ', 'GOOGL', 'GOOG', 'BRK.B', 'XOM', 'JPM', 'FB', 'AMZN', 'MSFT', 'AAPL']```

现在，打印股票代码就变得稍微容易一些了，你也不必记住字符串的名字！


print(VINIX[0])

C
print(VINIX[1])
MA

_稍后你将学习如何更高效地打印列表中的元素._

你甚至可以查看`VINIX`是否**包含**某一特定的股票<br>
像这样：

'GE' in VINIX

False

'GOOGL' in VINIX

True
```

我们将在本课后面再次研究华尔街的这个例子，以展示数据结构更多可使用的场景！
